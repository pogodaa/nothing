ПРЕДМЕТНАЯ ОБЛАСТЬ
------------------
Компания: ООО «ВелосипедДрайв» — магазин по продаже велосипедов.
Цель системы: получать актуальную информацию об остатках товаров на складах.

Тип решения: ДЕСКТОПНОЕ ПРИЛОЖЕНИЕ С ОКНАМИ (не веб-сайт).
База данных: SQLite (или другая СУБД по выбору, в ТЗ указан SQLite в списке ПО).
Стиль кода для Python: snake_case, не более одной команды в строке.


РОЛИ ПОЛЬЗОВАТЕЛЕЙ
------------------
1. ГОСТЬ
   - Вход без логина (кнопка перехода к товарам)
   - Просмотр списка товаров
   - БЕЗ поиска, фильтрации, сортировки
   - БЕЗ заказов, БЕЗ редактирования

2. АВТОРИЗОВАННЫЙ КЛИЕНТ
   - Вход по логину и паролю из БД
   - Просмотр товаров (как гость — без поиска/фильтра/сортировки)

3. МЕНЕДЖЕР
   - Вход по логину и паролю
   - Товары: просмотр + поиск + фильтрация + сортировка
   - Заказы: только ПРОСМОТР (кнопка «Заказы»)
   - НЕ может добавлять/редактировать/удалять

4. АДМИНИСТРАТОР
   - Полный доступ к товарам: просмотр + поиск + фильтр + сортировка + CRUD
   - Полный доступ к заказам: просмотр + CRUD
   - Может загружать/менять фото товаров


ОБЩИЕ ТРЕБОВАНИЯ К ИНТЕРФЕЙСУ (все модули)
------------------------------------------
- При запуске ПЕРВЫМ окном всегда идёт ОКНО ВХОДА
- После входа в правом верхнем углу — ФИО пользователя
- Из любой учётной записи — выход на главный экран (окно входа)
- Заголовок каждого окна соответствует назначению
- Логотип компании на главной форме (из ресурсов, не искажать)
- Иконка приложения (icon.ico / icon.png из ресурсов)
- Шрифт: Arial
- Цвета:
    Основной фон:        #FFFFFF (белый)
    Дополнительный фон:  #6A5ACD
    Акцент (кнопки):     #4B0082
    Скидка > 15%:        фон строки #483D8B


ДАННЫЕ ДЛЯ ИМПОРТА (Приложение 2, папка import)
-------------------------------------------------
Файлы:
  - Tovar.xlsx              — товары
  - user_import.xlsx        — пользователи (роль, ФИО, логин, пароль)
  - Заказ_import.xlsx       — заказы
  - Пункты выдачи_import.xlsx — адреса пунктов выдачи
  - 1.JPG … 10.JPG          — фото товаров
  - picture.png             — заглушка если нет фото
  - icon.ico / icon.png     — иконка приложения

Поля товара (Tovar.xlsx):
  Артикул, Наименование, Единица измерения, Цена, Поставщик, Производитель,
  Категория, Действующая скидка (%), Кол-во на складе, Описание, Фото

Поля пользователя (user_import.xlsx):
  Роль сотрудника, ФИО, Логин, Пароль

Поля заказа (Заказ_import.xlsx):
  Номер заказа, Артикул заказа (строка вида «А112Т4, 2, G843H5, 2»),
  Дата заказа, Дата доставки, Адрес пункта выдачи (номер/ссылка),
  ФИО клиента, Код для получения, Статус заказа

СТРАНИЦА 2–4 — МОДУЛЬ № 2: АЛГОРИТМ И ПРИЛОЖЕНИЕ
Время: 40 минут
================================================================================

ЧТО СДЕЛАТЬ:

1. БЛОК-СХЕМА АЛГОРИТМА
   - По ГОСТ 19.701-90
   - Формат: PDF

2. ПОДКЛЮЧИТЬ БД К ПРИЛОЖЕНИЮ
   - Список товаров берётся из базы данных

3. ОКНО ВХОДА (первый экран)
   - Поля: логин, пароль
   - Кнопка входа
   - Возможность перейти к товарам как ГОСТЬ

4. АВТОРИЗАЦИЯ
   - Логин/пароль проверяются по БД
   - После входа — доступ к модулям по роли
   - ФИО в правом верхнем углу
   - Кнопка выхода → окно входа

5. СПИСОК ТОВАРОВ (для всех ролей, с разным функционалом)
   Поля в карточке/строке товара:
     - Фото (или picture.png-заглушка)
     - Наименование товара
     - Категория товара
     - Описание товара
     - Производитель
     - Поставщик
     - Цена
     - Единица измерения
     - Количество на складе
     - Действующая скидка

   Макет карточки (стр. 4):
     [Фото] | Категория | Наименование
            | Описание | Производитель | Поставщик
            | Цена | Ед.изм. | Кол-во на складе     | Скидка

6. ПОДСВЕТКА СТРОК (обязательно):
   - Скидка > 15%  → фон строки #483D8B
   - Есть скидка   → старая цена перечёркнута, красная;
                     рядом итоговая цена чёрным
   - Кол-во = 0    → вся строка серым цветом

7. ОТЛАДКА
   - Приложение не должно падать
   - Скриншоты корректной работы → DOCX

ЧТО СДАТЬ:
  - PDF блок-схемы
  - Рабочее приложение (окно входа + список товаров)
  - DOCX со скриншотами

ПРИЛОЖЕНИЯ:

СТРАНИЦА 5–7 — МОДУЛЬ № 3: ИНТЕРФЕЙС, CRUD ТОВАРОВ
Время: 1 час 30 минут
================================================================================

ЧТО СДЕЛАТЬ:

1. НАВИГАЦИЯ
   - Переходы между окнами (вперёд/назад, кнопка «Назад»)
   - У каждого окна — свой заголовок

2. ОБРАБОТКА ОШИБОК
   - Сообщения об ошибках, запретах, предупреждениях
   - Окна с заголовком и пиктограммой (ошибка / предупреждение / информация)
   - Текст полезный: что не так и как исправить
   - Визуальные подсказки при вводе
   - Комментарии в коде только в неочевидных местах

3. ПОИСК (только менеджер и администратор)
   - Поисковая строка
   - По всем текстовым полям одновременно
   - В РЕАЛЬНОМ ВРЕМЕНИ (без кнопки «Найти»)

4. СОРТИРОВКА (менеджер и администратор)
   - По возрастанию и убыванию:
     * количество на складе
     * цена
     * действующая скидка
   - Параметры сортировки сохраняются при фильтрации и поиске

5. ФИЛЬТРАЦИЯ (менеджер и администратор)
   - Выпадающий список по размеру скидки:
     * «Все диапазоны» (первый пункт, сбрасывает фильтр)
     * 0 – 11,99%
     * 12 – 18,99%
     * 19% и более
   - В реальном времени, совместно с поиском

6. ДОБАВЛЕНИЕ / РЕДАКТИРОВАНИЕ ТОВАРА (только администратор)
   - Отдельное окно-форма
   - Переход: клик по товару = редактирование; кнопка «Добавить товар» = новый
   - Только ОДНО окно редактирования одновременно

   Поля формы:
     - Фото товара (заглушка picture.png)
     - Наименование
     - Категория (выпадающий список)
     - Описание
     - Производитель
     - Поставщик (выпадающий список)
     - Цена (дробная, >= 0)
     - Единица измерения
     - Количество на складе (>= 0)
     - Действующая скидка

   ID товара:
     - При добавлении: не показывается, auto = max(id)+1
     - При редактировании: только для чтения

   Фото:
     - Админ может добавить/заменить
     - Сохранять в папку приложения, в БД — путь к файлу
     - Ограничение размера: 300×200 пикселей
     - При замене — удалить старый файл

7. УДАЛЕНИЕ ТОВАРА (только администратор)
   - Нельзя удалить товар, если он есть в заказе

8. ОБНОВЛЕНИЕ СПИСКА
   - После добавления/редактирования/удаления список товаров обновляется
